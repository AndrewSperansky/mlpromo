

# Model Card — Promo Uplift Model

## 1. Общая информация

**Model name:** promo_uplift  
**Algorithm:** CatBoostRegressor  
**Task type:** Regression (promo uplift / sales prediction)  
**Owner:** Promo-ML  
**Current status:** Production (single active model)

---

## 2. Бизнес-назначение модели

Модель предназначена для:
- оценки эффекта промо-акций
- поддержки решений о запуске промо
- прогнозирования ожидаемых продаж с учётом скидок

Модель является **рекомендательной** и не заменяет бизнес-экспертизу.

---

## 4. Область применения и ограничения

### Применима:
- SKU с историей продаж
- стандартные промо-механики
- диапазоны цен и скидок, близкие к обучающим данным

### Не применима:
- новые SKU без истории
- нетипичные промо (разовые, ручные)
- экстремальные скидки вне обучающего диапазона

---

## 5. Используемые признаки (features)

| Feature | Описание |
|------|--------|
| price | Цена SKU |
| discount | Размер скидки (%) |
| avg_sales_7d | Средние продажи за 7 дней |
| promo_days_left | Дней до окончания промо |
| stock_level | Остатки на складе (если используется) |

Состав признаков контролируется и не должен изменяться без переобучения модели.

---

## 6. Обучающие данные

- Источник: корпоративное хранилище данных
- Тип данных: агрегированные табличные данные
- Период: исторические промо-кампании

Предобработка:
- очистка выбросов
- обработка пропусков
- нормализация не применяется (CatBoost)

## 8. Explainability (SHAP)

Для объяснения модели используется SHAP (TreeSHAP).

Артефакты:
- `shap_summary.json` — глобальная важность признаков
- `shap_values.npy` — SHAP значения на обучающих данных
- `shap_expected_value.json` — базовый прогноз модели

SHAP используется:
- для интерпретации решений
- для контроля деградации модели
- для аудита и доверия со стороны бизнеса

SHAP **не вычисляется в runtime**.

---

## 9. Drift monitoring

Отслеживаются два типа drift:

### Data drift
- изменение распределения входных данных
- контролируется с помощью PSI

### SHAP drift
- изменение вклада признаков в прогноз
- сигнализирует о логической деградации модели

Drift используется для принятия управленческих решений, а не для автоматического изменения модели.

---

## 10. Fallback и отказоустойчивость

При критическом drift:
- модель может быть отключена
- используется baseline-прогноз
- бизнес-процесс не останавливается

Fallback является обязательной частью системы.

---

## 11. SLA и эксплуатация

- модель используется через API
- SHAP и drift не считаются в runtime
- время ответа API стабильно и предсказуемо

Это обеспечивает соблюдение SLA.

## 12. Риски и допущения

- модель чувствительна к изменениям ценовой политики
- прогнозы имеют вероятностный характер
